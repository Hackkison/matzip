# PRD — 맛집 기록 & 공유 서비스

## 1. 문제 정의

맛집 사진 찍기를 즐기고 지인들에게 맛집 추천을 자주 하는 사용자가 겪는 불편함:

- 추천 맛집 정보가 사진첩, 지도 앱, 메시지 앱 등에 **분산 저장**되어 다시 찾거나 공유하기 번거롭다.
- 추천을 받은 사람의 후기가 다시 공유되지 않아 **추천 경험이 단절**된다.

---

## 2. 목표

맛집을 한 곳에서 **기록·정리·공유**할 수 있는 서비스를 만든다.

- 카테고리별 정리
- 사진과 기본 정보를 한눈에 확인
- 추천받은 사람이 후기를 남기고 다시 공유하는 선순환 구조

---

## 3. 대상 사용자

| 구분 | 설명 |
|------|------|
| 1차 사용자 | 맛집을 기록하고 추천하는 사람 (서비스 주인) |
| 2차 사용자 | 추천을 받아 방문하고 후기를 남기는 사람 |

---

## 4. 사용자 흐름

```
앱 진입
  └─> 지역 선택 (SVG 지도 UI)
        └─> 시/도 선택 → 시/군/구 단위 지도 모달 표시
              └─> 원하는 지역 클릭 → 선택 확정
                    └─> 해당 지역 맛집 리스트 탐색
                          └─> 원하는 식당 선택
                                └─> 식당 상세 페이지 확인
                                      └─> (선택) 후기 작성
```

> 지역 선택 → 리스트 → 상세 페이지 진입까지 **3화면 전환 이내** 완료 (모달 내 선택은 동일 화면)

---

## 5. 핵심 기능

### 5-0. 지역 선택 (SVG 지도 UI)
- 메인 화면에서 시/도 단위 지도 표시
  - **SVG 지도는 실제 대한민국 행정구역 형태**를 정확히 반영해야 함 (단순화된 임의 도형 사용 불가)
- 시/도 클릭 시 해당 시/도의 시/군/구 단위 지도 모달 표시
- 모달 내 구성
  - 상단: 선택한 시/도명 + 닫기(X) 버튼
  - 중앙: **SVG 지도 — 실제 시/군/구 행정구역 형태**로 표시, 클릭 시 해당 영역 강조(어둡게)
  - 하단: 전체선택 체크박스 + 선택 확정 버튼
- 선택 확정 시 해당 지역 맛집 리스트 화면으로 이동
- 참고 UI: 석구식 서비스의 지역 선택 모달 (시/도 클릭 → 시/군 지도 형태 드릴다운)
- **모든 17개 시/도에 대해 시/군/구 단위 SVG 지도 제공**

#### SVG 지도 데이터 구축 방침
- 공개 GeoJSON 데이터(southkorea/korea-maps 등) 기반으로 SVG 경로 생성
- Mapshaper 등으로 좌표 간소화하여 용량/품질 밸런스 확보
- 17개 시/도별 SVG를 별도 파일로 분리, 클릭 시 해당 파일만 lazy-load → 초기 로딩 최소화

### 5-1. 맛집 목록 조회
- 지역별 식당 리스트 표시
- 리스트 카드에 **이름, 카테고리, 위치, 대표 사진** 노출

### 5-2. 카테고리 분류
- 한식 / 중식 / 일식 / 양식 / 디저트 / 기타

### 5-3. 맛집 상세 정보 조회
- 식당 이름, 카테고리, 주소
- 지도 위치 확인 (지도 뷰 제공)
- 대표 사진 및 사용자 업로드 사진

### 5-4. 사진 업로드
- 사용자가 방문 시 찍은 사진 첨부 가능

### 5-5. 후기/평가 작성
- 리뷰를 **두 가지 출처로 구분** 표시:
  - **카카오 리뷰**: 카카오맵에서 가져온 외부 리뷰 (읽기 전용, 출처 라벨 표시)
  - **사용자 리뷰**: 서비스 내 사용자가 직접 작성한 리뷰 (수정·삭제 가능)
- 상세 페이지에서 탭 또는 섹션으로 분리 (예: "카카오 리뷰 | 우리 리뷰")
- 평점도 각각 별도 집계 — 카카오 평점 / 서비스 내 평점 구분 표시
- 사용자 리뷰 작성 시 별점 + 텍스트 + 사진 첨부

### 5-6. 맛집 등록
- 가게명 입력 시 **Kakao Local API 키워드 검색**으로 자동완성 (category_group_code=FD6 필터)
- 선택 시 가게명, 주소(지번/도로명), 카테고리, 전화번호, 좌표 자동 입력
- 카테고리 매핑: Kakao 카테고리(음식점 > 한식 > 국밥) → 서비스 카테고리(한식)로 변환
- 대표 사진 첨부 (선택)
- 등록 후 해당 식당 상세 페이지로 이동
- 디바운싱 300ms 적용, 최소 2글자 이상 입력 시 검색 호출

### 5-7. 검색
- 식당명, 지역명, 카테고리로 통합 검색
- Supabase Full Text Search 또는 `ilike` 쿼리 기반

## 10. 기술 스택

### 선택 스택: Next.js + Supabase + Kakao Maps + Vercel

| 영역 | 기술 | 선택 이유 |
|------|------|----------|
| 프론트엔드 | Next.js + Tailwind CSS | SSR/SSG로 모바일 3초 로딩 달성, HR 앱과 동일 프레임워크 |
| UI 컴포넌트 | shadcn/ui | HR 앱(Geist 폰트, 미니멀 기업 디자인)과 일관된 스타일 |
| 데이터베이스 | Supabase (PostgreSQL) | 맛집·리뷰 데이터 저장 |
| 인증 | MVP: Supabase Auth (독립) → 이관 시 HR 앱 JWT 공유로 전환 | MVP 우선 출시 후, HR 앱 이관 단계에서 도메인·CORS·JWT 검증 방식 재검토 |
| 파일 스토리지 | Supabase Storage | 방문 사진 업로드 |
| 지도 | Kakao Maps API | 한국 주소 검색·지도 표시 정확도 |
| 배포 | Vercel | Next.js 궁합 최적, HR 앱과 동일 인프라 |

### 디자인 가이드 (HR 앱 참고)

| 항목 | 값 |
|------|-----|
| **주요 브랜드 컬러** | `#1B4332` (포레스트 그린 — HR 앱 메인 배경색 기준) |
| **보조 컬러** | `#ffffff` (흰색 텍스트/배경) |
| **텍스트 on 그린** | `#ffffff` |
| **일반 배경** | `#ffffff` |
| **서브 배경** | `#f5f5f5` |
| **버튼/강조** | `#1B4332` (Primary), `#e5e5e5` (Secondary) |
| 폰트 | Geist (HR 앱과 동일) |
| 스타일 | 미니멀 기업용 — 불필요한 장식 없이 정보 중심 |
| 컴포넌트 | shadcn/ui 기반 (HR 앱 UI 패턴 일치) |

> HR 앱 로그인 화면의 짙은 포레스트 그린(`#1B4332`)을 브랜드 컬러로 사용. 버튼·헤더·강조 요소에 적용하고 일반 콘텐츠 영역은 흰색 배경 유지.

### HR 앱 연동 전략

| 항목 | 방식 |
|------|------|
| 인증 | **MVP: Supabase Auth 독립 인증** → HR 앱 이관 시 JWT 토큰 공유로 전환 (이관 시 도메인 구조, CORS, SameSite 쿠키 정책, JWT 검증 방식 재검토 필요) |
| 사용자 정보 | HR 앱 API에서 사용자 이름·부서 가져와 리뷰에 표시 |
| 진입점 | HR 앱 내 메뉴 또는 링크로 맛집 서비스 연결 |
| 도메인 | hr.digiwood.co.kr과 동일 도메인 하위 경로 또는 서브도메인 권장 |
